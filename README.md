## Подготовка к работе
1. Склонировать репозиторий
2. Запустить docker
```bash
docker compose up -d
```
3. База данных наполняется самостоятельно через сервис mongo-seed 
4. Перейти по адресу http://localhost/ и на странице должны быть продукты.
5. На странице http://localhost/login/ можно авторизоваться.
6. Админка находится по адресу http://localhost/admin/

## Сводка проведённых работ по обеспечению безопасности

Проведён аудит безопасности — покрыты все слои приложения: backend, frontend и инфраструктура. Исправлено **33 уязвимости**.

### Исправления по направлениям защиты

**Аутентификация и управление сессиями**
- Замена MD5 на bcrypt (12 раундов) для хеширования паролей
- Вынос JWT-секретов и CSRF-секрета в `requireEnv()` с fail-fast в production
- Сокращение срока жизни токенов: access — 15 мин, refresh — 7 дней
- Явное указание алгоритма HS256 при подписи/верификации JWT
- Защита от replay-атак: refresh-токен одноразовый, проверка наличия хеша перед удалением
- Cookie `secure` — динамически в зависимости от NODE_ENV
- Исправлена логика авторизации создания заказа

**Контроль доступа**
- Устранение IDOR на эндпоинтах /customers — добавлены roleGuardMiddleware
- Защита от массового назначения (Mass Assignment) в `PATCH /auth/me` и `PATCH /customers/:id` через Joi `stripUnknown: true`
- Удаление кастомного serveStatic middleware — замена на `express.static`
- Санитизация имён загружаемых файлов через `sanitizeFileName()`
- Глобальная CSRF-защита (double-submit cookie) для всех state-changing эндпоинтов
- Удалён мёртвый `User.findById` с проекцией через req.body

**Защита от инъекций**
- NoSQL-инъекция в `getOrders`: Joi `valid(...StatusType)` + type guard
- ReDoS: `escapeRegExp()` во всех search-обработчиках + `Joi.string().max(200)`
- Валидация sortField/sortOrder через allowlist
- Валидация тела запроса на login/register через Joi
- Защита от stored XSS: DOMPurify.sanitize() на комментариях заказа (backend + frontend)
- Частичное исправление валидации fileName при обновлении продукта

**Безопасность инфраструктуры**
- Подключение `helmet` с CSP-политикой
- Rate limiting: 5 запросов/5 с на auth, 100/5 с на остальные (по .env.example)
- Ограничение размера тела запроса — 1 MB
- Запрет загрузки SVG-файлов, лимит 10 MB на файл
- Порт MongoDB и порт backend больше не экспонируются на хост
- Смена `error_log debug` → `error_log warn` в nginx
- Rate limiter kill-switch доступен только в development
- Ограничение пагинации продуктов: `Math.min(limit, 100)`

**Логирование и обработка ошибок**
- Все `console.log` заменены на winston-логгер
- movingFile переписан на `fs/promises.rename()` с try-catch
- Обработчик ошибок в production не логирует стектрейсы

**Управление зависимостями**
- Обновление `react-router-dom` ^6.23.1 → ^6.30.3
- Устранение 14 уязвимостей npm-зависимостей; замена celebrate на joi

**Криптография и секреты**
- Замена строкового сравнения паролей на `bcrypt.compare()` — защита от timing-атаки
- Очистка seed-данных: удалены устаревшие хеши refresh-токенов
